# 趋势跟踪 Agent PRD（简要版）

## 1. 背景与目标

- **背景**：参加 FLOW AI 投资交易 Agent 24h 黑客松（2026年7月4-5日，北京），Builder Track 赛道。要求 24 小时内做出端到端跑通闭环的交易 Agent，依据 hello-agents 的 Agent Loop 理论构建。
- **目标**：构建一个"投资想法→策略生成→执行→风控→反馈"全闭环自主运行的趋势跟踪 Agent，可现场跑测、可演示、可观测。
- **范围**：24h MVP，单标的、模拟撮合、不接实盘资金。

## 2. 产品定位

| 维度 | 内容 |
|---|---|
| 产品形态 | 混合式 LLM 交易 Agent（底层反应式风控 + 高层规划式决策） |
| 目标用户 | 趋势交易散户 / 小型量化交易者 |
| 核心价值 | 让"看趋势下单"这件事由 Agent 自主闭环完成，用户只给投资想法和风险预算 |
| 与普通工具的区别 | 自主跑通闭环，非人驱动单步；风控硬规则不经过 LLM |

## 3. 核心流程（Agent Loop）

感知（拉K线/算指标）→ 思考（LLM 判断趋势+选工具）→ 行动（下单/调仓）→ 观察（成交回报+浮盈）→ 循环

每轮强制输出 `Thought / Action / Observation` 三段式，作为"是真 Agent"的可观测证据。

## 4. 功能模块

| 模块 | 职责 | 实现要点 |
|---|---|---|
| 数据/指标层 | 传感器 | K线 API + MA/ATR/ADX 计算 |
| 工具库 | 执行器 | `get_kline` / `calc_position` / `place_order` / `check_risk` 等函数 |
| LLM 决策层 | 思考 | system prompt 强制三段式输出 |
| 风控层 | 反应式底层 | 纯代码硬规则，止损/限仓先于下单，**不过 LLM** |
| 反馈/记忆层 | 观察 | 成交回报 + 决策日志，供下一轮与复盘 |
| 可观测界面 | 产品完整度 | 展示每轮决策三段式 + 持仓盈亏曲线 |

## 5. 优先级

| 优先级 | 模块 | 说明 |
|---|---|---|
| P0 | 数据指标层 + 工具库 + LLM 决策层 + 风控层 | 闭环最小可跑通，缺一不可 |
| P0 | 模拟撮合 + 决策日志 | 现场跑测与复盘的证据 |
| P1 | 可观测界面 | 演示效果，评委可见度 |
| P1 | 记忆层（历史趋势判断） | 多轮连贯性 |
| P2 | 多标的 / 多周期 | 24h 外的扩展，比赛不追求 |

## 6. 验收标准（对齐 Builder Track 三维度）

- **产品完整度**：闭环四环节均有实现，可观测界面能看到 Agent 每轮在想什么、调了什么、下了什么单、风控为何触发。
- **独立运行能力**：启动后用户只给投资想法+风险预算，Agent 自主跑完一轮感知到反馈，无需逐步喂 prompt。
- **真实用户价值**：能解决"盯盘看趋势→带风控下单"这一具体痛点，非为比赛编造的通用助手。
- **底线红线**：能真实下单并成交（模拟），非只聊天、非只总结研报、非漂亮 Demo。

## 7. 待确认事项

2. **LLM 选型**：通义千问 / OpenAI / 其他——影响成本与稳定性
